# 国际化（UI 文案与语言）

> **本文档**：用户可见文案、locale、`useUiCopy` / 场景 copy、与 `constant` 及领域数据的边界、`lang` 与新增语言流程。  
> **相关**：[文档索引](./README.md) · [代码组织](./code-organization.md) · [状态与偏好](./state-management.md)

---

本文约定：**用户界面**上可见的自然语言字符串如何存放、如何接入组件，以及与 `constant.ts`、领域数据的边界。

## 与 `constant.ts` 的分工

| 内容 | 放在哪里 |
|------|----------|
| 路由 path 片段、稳定 id、功能开关、魔法数字、阈值 | `constant.ts`（或 feature 内等价命名文件） |
| 按钮/标题/提示/空态/aria 标签等 **用户可读文案**（需随语言切换） | **`shared/lib/i18n`** 下的 copy 表，或下文所述的 **场景 copy 文件** |
| 文章正文、目录标题、mock 文档内容（来自数据层） | **数据 / repository / fixtures**，不是 i18n 表；若需多语言内容，由内容模型与数据源决定，不在本文「UI copy」范围内 |

原则：**不在 TSX 里写中英混排的业务句子**；测试里为断言需要的字面量除外（见 `testing-strategy.md`）。

## 语言枚举与默认值

- **类型与列表**：`AppLocale`、`APP_LOCALES` 定义在 `shared/lib/i18n/locale.types.ts`（例如 `zh-CN`、`en-US`）。
- **默认语言**：`DEFAULT_LOCALE` 在同一文件；产品默认值变更时只改此处并评估已有用户偏好存储。
- **校验**：对外部或存储读出的字符串，用 `isAppLocale()` 再收窄类型，避免非法 locale 流入 UI。

## 主产品 UI 文案（`app` + `features`）

- **数据表**：`shared/lib/i18n/ui-copy.ts` 内维护 `Record<AppLocale, UiCopy>`（或等价结构），**类型 `UiCopy` 与词条同文件维护**，保证新增 key 时类型与双语同时补齐。
- **读取**：
  - 组件内使用 hook **`useUiCopy()`**（实现见 `shared/lib/i18n/use-ui-copy.ts`），从当前偏好中的 `locale` 解析文案。
  - 无 React 上下文时（如少量纯函数）使用 **`getUiCopy(locale)`**。
- **主题等与语言相关的无障碍文案**：例如 `getThemeToggleAriaLabel(locale, theme)`，与 locale 强相关时放在 `ui-copy.ts` 或与其并列的 helper，避免在组件里拼接句子。
- **日期**：展示用格式化使用 `shared/lib/date/format-date.ts` 等工具，**传入当前 `AppLocale`**，与 `Intl.DateTimeFormat` 等保持一致。

## 文档站 / 场景专用文案（`site`）

- **何时单独建表**：某入口（如 `site/overview`）有大量 **仅在该场景出现** 的说明、演示标签、侧栏标题，与主产品 `UiCopy` 结构差异大时，使用 **场景级 copy 文件**（例如 `site/overview/overview-copy.ts`），同样维护 `Record<AppLocale, …>` 与 TypeScript 类型。
- **读取**：提供 **`useXxxCopy()`** / **`getXxxCopy(locale)`**，内部仍从 **`usePreferences()`** 取 `locale`，与主产品一致。
- **与主产品复用**：若文案语义与主产品相同（如「语言切换」按钮、`kind` 标签、阅读时长单位），应 **`useUiCopy()`** 或 `common` 段已有函数，**禁止**在场景里再写一套「中 / EN」或与 `ui-copy` 矛盾的字符串。

## 语言偏好与 DOM

- **读写**：语言与主题等一并走偏好层与 repository（见 `state-management.md`、repository 偏好 API）。
- **`document.documentElement.lang`**：须与当前 `AppLocale` 一致；通过已有 DOM 适配（如 `useDocumentElementLanguage`）更新，**不在业务组件里直接改 `document.lang`**。

## 新增词条流程

1. 在对应 copy 文件的 **类型** 上增加字段（或扩展函数签名），避免「只有运行时字符串没有类型」。
2. 在 **`zh-CN` 与 `en-US`（及已有全部 locale）** 下同时填入译文，避免半套语言。
3. 在组件中改为引用 `copy.xxx`，删除 TSX 内硬编码。
4. 若词条带 **复数、插值、性别** 等，用 **函数** `(args) => string` 写在 copy 表内，不把拼接规则散落在组件里。

## 禁止与反例

- 在顶栏、按钮上写死 `"EN"` / `"中"` 等与 **`localeToggle`** 重复的切换文案，而不走 `useUiCopy()`。
- 维护两套含义相同的中英文字典（主产品与文档站各一份）且长期不同步。
- 把 **领域数据**（如内容标题、标签 slug）误当成 UI copy 写进 `ui-copy.ts`；应来自节点 / 文档模型。

## 新增一种界面语言（例如 `ja-JP`）

1. 扩展 `APP_LOCALES` 与 `AppLocale`。
2. 在 **`ui-copy.ts`（及所有场景 `*-copy.ts`）** 为新区块补全对象；缺省语言不可用时应有明确策略（例如回退到 `en-US`），并在实现处写清。
3. 回归与语言相关的偏好持久化与 `lang` 属性。
4. 产品层确认 **默认 locale** 与内容源语言策略。

以上步骤涉及「未约定的新抽象」时，先按 `README.md` Repo Consensus 报备。
